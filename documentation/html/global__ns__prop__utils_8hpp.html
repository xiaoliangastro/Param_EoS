Find non-rotating maximum supported mass of an EoS.

The challenge is the "noise" or none monotonicity property, the solution is:

  1. A bigger guard to check again, 2. Judge whether a step before is larger. error: ~0.005 M_sun
    Parameters
    hcPointer to the central enthalpy, return -1 if finding error.
    M_maxPointer to the maximum mass, return 0 if finding error.
    h_startWhere to start finding the maximum mass.
    compare_ftype_is_lessFind mimimum or maximum, set to true if maximum is wanted.
    jump_sizeSet the initial jump size of h to find the maximum mass.
    check_okWhether to set the (mtov_min, mtov_max) constraint or not.
    Return values
    boolwhether finding process correctly worked.

Find maximum mass in the case that there are two branches of M-R (phase transition of EoS).

Need the transition density and strenth to find three critical structure and output the corresponding hc and mass.

Parameters
h1Pointer to the central enthalpy of first peak in M-R graph, return -1 if finding error.
h2Pointer to the central enthalpy of first valley in M-R graph, return -1 if finding error.
h3Pointer to the central enthalpy of second peak in M-R graph, return -1 if finding error.
h_maxPointer to the central enthalpy of maximum mass in M-R graph, return -1 if finding error.
m1Pointer to the mass corresponding to h1, return 0 if finding error.
m2Pointer to the mass corresponding to h2, return 0 if finding error.
m3Pointer to the mass corresponding to h3, return 0 if finding error.
m_maxPointer to the mass corresponding to h_max, return 0 if finding error.
rho_trThe starting mass density of phase transition, to boost the finding process.
drhoThe length of mass density of phase transition, to boost the finding process.
Return values
boolwhether finding process correctly worked.

◆ get_mrl()

double * get_mrl ( double  hc)

Python interface function for integrating the whole structure of the star.

Give h_c, get M/M_sun, R/km and L, which are stored in global variable double *mrl_result.

Attention
Use this for python control only, because double* is convenient for python, and global variable is needed,
which may cause confuse problem if used in c++ main, so please use int_whole_star instead for c++, because that do not need to return global variable.
Parameters
hcenthalpy at the center of the compact star.
Return values
double*A double array of [mass, radius, tidal-deformability].
